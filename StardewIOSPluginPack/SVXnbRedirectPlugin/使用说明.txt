星露谷 iOS 新增 XNB 重定向插件

用途：
让游戏读取 Content 下新增或覆盖的 XNB 时，优先从 Documents/CustomContent 读取。
同时把 Documents/CustomContent 下的新增 XNB 资源名动态加入游戏的资源检测清单，绕过 iOS 1.6.15 新增资源名找不到的问题。

目录示例：
游戏请求 Content/Maps/wind_valley_tiles.xnb
你放置 Documents/CustomContent/Maps/wind_valley_tiles.xnb

游戏请求 Content/Buildings/Wind Valley Barn.xnb
你放置 Documents/CustomContent/Buildings/Wind Valley Barn.xnb

使用步骤：
1. 用 GitHub Actions 或 macOS 编译 SVXnbRedirectPlugin.dylib。
2. 用全能签导入 dylib 后重签游戏。
3. 启动游戏一次。
4. 查看 Documents/sv_xnb_redirect.log，确认插件已加载。
5. 按 Content 相同目录结构把新增 XNB 放入 Documents/CustomContent。

日志判断：
看到 SVXnbRedirectPlugin loaded 表示 dylib 已加载。
看到 mono api loaded=yes 表示可以调用 Mono 运行时。
看到 manifest patch after ... added/tried ... asset names 表示已尝试把新增 XNB 加入资源检测清单。
看到 open redirect/fopen redirect 表示实际读取已重定向到 CustomContent。

注意：
地图内部引用素材时，建议写 Maps/xxx，不要写 .xnb 或 .png。
路径大小写必须完全一致。
如果游戏启动时 CustomContent 里还没有文件，请放入文件后重启游戏，让插件重新扫描。
